# Content Research Foundation (India Dropshipping MVP)

This note captures what to personalize and why, based on a research sweep of official guidance pages, marketplace onboarding pages, and compliance explainers.

## Core personalization factors we should collect

1. Business type (individual/proprietorship/partnership/LLP/private limited)
2. Sales model (marketplace only, own website only, or both)
3. Product category (general, food, beauty, electronics, fashion)
4. Operating state
5. GST status (available/not yet)
6. Imported products (yes/no)
7. Pre-packaged goods (yes/no)
8. Primary channel (Amazon/Flipkart/Meesho/Shopify)

## Why these factors matter

- **Business type** changes mandatory document sets and authorized signatory requirements.
- **Sales model** changes immediate GST/compliance urgency (marketplace-first usually stricter from day 1).
- **Product category** changes compliance depth (food and electronics usually need extra checks).
- **Operating state** changes practical registration + address-proof workflow and local compliance execution.
- **Imports** can trigger importer labeling/compliance responsibilities.
- **Pre-packaged** products trigger packaged declarations responsibilities.
- **Primary channel** changes onboarding sequence and operational checklist.

## High-signal source themes used

- Amazon seller documentation and FAQ pages emphasize PAN/GST/bank onboarding consistency.
- Meesho supplier onboarding pages show GSTIN-or-UIN paths and pickup address consistency constraints.
- GST documentation checklists outline entity-wise required documents.
- FSSAI documentation emphasizes central licensing obligations for relevant e-commerce food operations.
- Legal metrology packaged rules emphasize mandatory online declarations for packaged commodities.
- BIS explainers emphasize category-based electronics compliance checks and labeling/registration expectations.

## Content architecture we should follow on each step page

1. Plain-language summary (no legal jargon wall)
2. Why this page is personalized (show input -> impact mapping)
3. Documents to keep ready (entity/category-specific)
4. Checklist in execution order
5. Mistakes to avoid (based on real onboarding rejection patterns)
6. Third-party options (when to use + what to ask, so user is not exploited)

## Guardrails for writing content

- Use simple language first; legal terms only with plain translation.
- Separate "must-do now" vs "later optimization".
- Do not present legal/tax guidance as final advice; prompt user to confirm with CA/professional for edge cases.
- Keep content action-first: every section should end in a practical next action.
